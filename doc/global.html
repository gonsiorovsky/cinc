define(module_name, deps, module)

AMD define emulation
IMPORTANT: This is not a full-featured AMD implementation. It does not have the ability to load external module. This function just appends an AMD module to the cinq storage.
NOTE: This function is only available when there is no other AMD realizations found.
Parameters:
Name Type Description
module_name string A unique name which will identify the module. cinq does not support anonymous modules, so they will be ignored.
deps Array A list of module's dependencies. For each item in this list function use will be applied and the results will be passed as arguments to the module's constructor (if any)
module Object | Function A module object or a constructor

require(deps, callback)

AMD require emulation
IMPORTANT: This is not a full-featured AMD implementation. It does not have the ability to load external module. This function just extracts AMD modules from the cinq storage and executes the callback.
NOTE: This function is only available when there is no other AMD realizations found.
Parameters:
Name Type Argument Description
deps Array A list of previously defined modules to include.
callback Function <optional>
 A function that will be called after modules are included successfully. This function will be passed loaded module objects as arguments.
